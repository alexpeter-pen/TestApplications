C# — Server, dinamička logika
Gde se koristi: Na serveru (.NET) za poslovnu logiku, validacije, servise, repozitorijume.
Dinamički generiše odgovore, poziva bazu i servise.
// Service primer
public interface IClock { DateTime Now(); }
public class Clock : IClock { public DateTime Now() => DateTime.UtcNow; }
Controller — MVC ulazna tačka (server)
Gde se koristi: Na serveru; prima HTTP zahtev, orkestrira Model/Servise i vraća View ili JSON. 
// ASP.NET MVC 5 (Framework 4.6.1)
public class PrognozaController : Controller
{
  public ActionResult Prikaz(string grad = "Beograd")
  {
    var vm = PrognozaRepoza.GetByCity(grad);
    return View(vm); // vraća Razor pogled
  }
}
Model — Podaci & pravila (server)
Gde se koristi: Na serveru; nosi podatke i invarijante domena, često mapiran na bazu. 
public class Prognoza {
  public string Mesto { get; set; }
  public string NazivMesta { get; set; }
  public int MinTemperatura { get; set; }
  public int MaxTemperatura { get; set; }
  public string Vreme { get; set; }
}
Razor — Server templating → HTML
Gde se koristi: Na serveru; meša C# sa HTML-om i generiše statički HTML odgovor. 
@model Prognoza
<h2>@Model.NazivMesta</h2>
<p>Min: @Model.MinTemperatura, Max: @Model.MaxTemperatura</p>
LINQ — Upiti nad kolekcijama (server)
Gde se koristi: Najčešće na serveru; filtriranje/projekcije nad kolekcijama ili IQueryable. 
var topli = prognoze.Where(p => p.MaxTemperatura >= 20)
                    .OrderBy(p => p.NazivMesta)
                    .ToList();
Database — Trajno skladište (server)
Gde se koristi: Na serveru; SQL/NoSQL, transakcije, indeksiranje. 
-- SQL primer
SELECT NazivMesta, MinTemperatura, MaxTemperatura
FROM Prognoza
WHERE MaxTemperatura >= 20
ORDER BY NazivMesta;
Web Service — API endpoint (server)
Gde se koristi: Na serveru; izlaže JSON/XML za klijente i druge servise. 
// Web API 2 (Framework 4.6.1)
[RoutePrefix("api/prognoza")]
public class PrognozaApiController : ApiController
{
  [HttpGet, Route("{grad}")]
  public IHttpActionResult Get(string grad)
  {
    var p = ProgRepo.GetByCity(grad);
    if (p == null) return NotFound();
    return Ok(p);
  }
}
XML — Format razmene (server & klijent)
Gde se koristi: Kao konfiguracija, ugovor, ili dataset; često čuvan/obrađen na serveru. 
<Prognoza>
  <NazivMesta>Niš</NazivMesta>
  <MaxTemperatura>10</MaxTemperatura>
</Prognoza>
HTML — Struktura (klijent, statično)
Gde se koristi: Na klijentu; kostur stranice i semantika. 
<section>
  <h2>Prognoza</h2>
  <div class="karta">Beograd — vedro</div>
</section>
CSS — Stil (klijent, statično)
Gde se koristi: Na klijentu; izgled, raspored, animacije. 
.karta{ padding:8px 12px; background:#0b1020; color:#e6eefb; border-radius:10px }
JavaScript — Dinamika UI (klijent)
Gde se koristi: Na klijentu; interakcije, validacije, pozivi API-ju. 
// Učitavanje sa API-ja i popuna liste
fetch('/api/prognoza/Beograd')
  .then(r => r.json())
  .then(p => document.querySelector('.karta').textContent =
        p.NazivMesta + ' — ' + p.Vreme);
View — UI sloj (server → klijent)
Gde se koristi: Na serveru (Razor) generiše HTML, koji je na klijentu statičan dok JS ne interveniše. 
@model IEnumerable<Prognoza>
<ul>
@foreach(var p in Model){
  <li>@p.NazivMesta — @p.Vreme (@p.MinTemperatura–@p.MaxTemperatura)</li>
}
</ul>
